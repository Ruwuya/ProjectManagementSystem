This project contains the domain layer of the Project Management System.
It is responsible for defining the core business logic and rules of the application.
The domain layer includes:
- Entities: Represent the main objects in the system, such as Project, Task, User, etc.
- Value Objects: Represent concepts that do not have a unique identity, such as TaskStatus, Priority, etc.
- Domain Services: Contain business logic that does not naturally fit within an entity or value object.
- Domain Events: Represent events that occur within the domain, such as TaskCompleted, ProjectCreated, etc.
The domain layer is designed to be independent of any specific technology or framework,
allowing it to focus solely on the business logic and rules of the application.
It serves as the foundation for the other layers, such as the application layer and the infrastructure layer,
which will interact with the domain layer to implement the overall functionality of the system.
